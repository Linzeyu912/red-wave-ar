# 权利与风险登记

## 1. 当前权利结论

本轮未发现任何图片、视频、历史照片、地图或文章具有可直接用于商业/公开 App 的开放许可。因此：

- 仓库只保存文字化观察、来源元数据和链接；
- 不保存网页图片、视频、视频截图、文章 PDF 或展板扫描件；
- 不把来源图直接用于贴图、展板、触发图、缩略图、宣传海报或 AI 重绘；
- 模型材质、地图、音效和 UI 图形必须原创或来自许可清晰的资源；
- 用户提供素材也必须登记权利，不能因为“由用户发来”就默认可公开使用。

## 3. 素材权利登记字段

后续每份外部素材至少记录以下字段，可进入单独 CSV 或清单：

| 字段 | 示例/说明 |
|---|---|
| `asset_ref_id` | `USR-S1-001` |
| `original_filename` | 用户提供的原文件名 |
| `provider` | 提供者姓名/机构 |
| `creator` | 摄影、作者、制图者 |
| `source_url_or_archive` | 原始链接或馆藏出处 |
| `creation_date` | 拍摄/出版日期 |
| `subject` | 电台正面、院落门楼等 |
| `location` | 拍摄地点 |
| `rights_holder` | 已知权利人 |
| `allowed_research` | 是否允许内部研究 |
| `allowed_app_display` | 是否允许 App 内展示 |
| `allowed_derivative_model` | 是否允许据此制作衍生 3D |
| `allowed_promotion` | 是否允许宣传使用 |
| `attribution_required` | 署名格式 |
| `expiry_or_scope` | 时间、地区、设备或活动限制 |
| `people_visible` | 是否含可识别人物 |
| `review_status` | 待审/通过/拒绝 |

## 4. 对外表述安全线

### 可以使用

- “依据公开史料设计的虚拟主题展馆”；
- “场景化复原设备”；
- “路线示意”；
- “现纪念馆由关帝庙改建，战时相关院落位于村西”；
- “1941 年体系成立，1943 年冬涧沟村建立秘密电台”。

### 需要证据或授权后使用

- “馆藏原件数字孪生”；
- “1:1 复原”；
- “官方授权数字展馆”；
- “唯一准确路线”；
- “首次公开的真实密码/电文”；
- “由某历史人物真实录音/形象还原”。

## 5. 发布前权利检查

- [ ] 每张图、每段音频、每个字体、每个第三方模型均有来源和许可记录。
- [ ] 没有把网页截图、视频帧或历史照片嵌入 GLB/纹理/APK。
- [ ] 原创路线图的底图和字体许可清晰。
- [ ] 人物姓名、肖像和名单经过项目负责人审核。
- [ ] 所有“复原”“原件”“授权”“1:1”字样逐项核验。
- [ ] 对外附件只写实际完成和有证据支持的效果。
