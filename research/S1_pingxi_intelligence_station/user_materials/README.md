## 2. 当前许可口径

用户主动把两张图片提供给本项目并要求整理素材包，因此当前允许：

- 在本项目内部查看和研究；
- 提取普通灰砖、瓦檐、石基、门洞纵深、材质粗糙度等不受特定作品表达限制的高层视觉特征；
- 用于检查原创模型是否符合 S1 已冻结的风格方向。

当前仍未确认：

- 摄影者和原始权利人；
- 是否允许把照片本身放入 App；
- 是否允许加入 ARCore/识图数据库；
- 是否允许用于触发卡、宣传海报、答辩材料或公开仓库；
- 第二张现馆浮雕作品是否允许被完整复制、重建或制作衍生作品。

在上述状态未关闭前，两张图均不得复制到 `app/`、`modeling_delivery/`、Blender 打包文件、GLB extras、纹理目录或公开演示附件。

## 3. 两张图片的使用规则

### USER-VIS-001 当前纪念馆入口

- 原文件名：`6d7e3f423df289cfa6e4eff77c35859e.jpg`
- 本地文件：`local_pending/USER-VIS-001_current_entrance.jpg`
- 尺寸：1440 × 996，约 202 KiB；无可用 EXIF 作者、时间或设备字段。
- 可提取：灰砖与石基关系、瓦檐层次、暗红木门框、门洞纵深、院落与山体的连续视线。
- 必须排除：国旗、金属栏杆、摄像头、真实牌匾与楹联、现代导视、现馆精确尺寸和 1:1 立面。
- 识图评价：有砖缝和文字等稳定特征，但天空面积较大、旗帜会移动、透视角度偏斜；不是 S1 首选识图图案。

### USER-VIS-002 当前纪念馆浮雕墙

- 原文件名：`微信图片_20260706194753_382_1.jpg`
- 本地文件：`local_pending/USER-VIS-002_relief_wall.jpg`
- 尺寸：5724 × 4032，约 9.01 MiB；无可用 EXIF 作者、时间或设备字段。
- 可提取：灰砖色差、瓦当排列、粗石基拼接、单色浮雕的粗糙度层级；可帮助理解现馆如何把路线、人员与通信主题组织在同一叙事面。
- 必须排除：浮雕人物姿态、服装、地图路线、地点文字、题字、烽火台、马队和完整构图；这些都是现代艺术化展陈，不能作为战时史料或直接重建对象。
- 识图评价：细节丰富、非重复特征多，若线下实际扫描对象就是该浮雕墙，它是比入口照片更强的技术候选；进入识图数据库前仍需权利确认、透视校正和 `arcoreimg eval-img` 实测。

## 4. 给建模模型的读取规则

1. 先读 `../style_reference_board.md`，再打开两张本地图；不能让图片反向推翻已冻结的风格和史实边界。
2. 原图只能作为屏幕旁参考，不得导入 Blender 后随工程打包。
3. 不得使用照片投射、照片建模、AI 图生 3D、图生图或逐轮廓描摹复制牌匾/浮雕。
4. 风格图中只允许出现原创的灰砖、石基、旧木、抽象山体/屋面和现代信息线。
5. 在 `style_review_report.md` 中分别写明每张图“采用了什么高层特征、主动排除了什么具体内容”。

## 5. 权利确认后怎么升级

- 仅允许内部建模：维持当前目录和忽略规则，不复制到运行包。
- 允许 App 展示：另建经过裁切、色彩和隐私检查的发布衍生文件，并登记用途、署名、分辨率和授权证据。
- 允许现场识图：制作专用识图版本，执行透视校正、裁切、压缩与 `arcoreimg` 测试；原图仍保留在素材档案，不直接替换。
- 允许宣传：单独确认宣传渠道、期限、署名和是否允许裁切，不与 App 展示许可混用。
