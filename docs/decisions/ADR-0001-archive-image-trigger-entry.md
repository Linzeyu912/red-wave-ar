# ADR-0001：归档 ARCore 触发图入口（F2），仅保留二维码 + 手动列表

- **状态**：Accepted（归档，可重启）
- **日期**：2026-07-20
- **决策者**：项目负责人（李雨霏）确认，技术负责人（林泽羽）记录
- **影响范围**：计划书 §1.2、§2.1（F2）、§3.2、§5.3、§6.17、§10.1、§13（M3D-07）、§15.3（MT-001）

## 背景

计划书 v1.2 设计了三个入口：
- F1 二维码（ZXing）
- F2 触发图（ARCore Augmented Images）
- F3 手动列表

M0 落实目标机型时确认：**主展示机 nubia Z70 Ultra（NX736，Android 15）不支持 ARCore**——未出现在 Google ARCore 认证设备列表（[developers.google.com/ar/devices](https://developers.google.com/ar/devices)），第三方实测（Android Police 评测）确认 Google Play Services for AR 不工作。nubia 旗舰（Z60/Z70 Ultra）历史上均未通过 ARCore 认证。

这意味着 F2 触发图入口在主展示机上不可用。备用机尚未落实，且即使落实，触发图入口的真机验收（计划书 §10.1、MT-001「10 次至少识别 9 次」）依赖 ARCore 认证机型 + 五张触发卡印刷件 + 评分 ≥75 的训练图——成本高、周期长。

## 选项

1. **归档 F2，仅保留 F1 + F3**（本决策）
   - 删除原生 App 对 ARCore 的依赖；
   - 二维码 + 手动列表覆盖所有场景，主展示机完全可用；
   - 触发图入口的代码与触发卡资产暂不生产，后续按需重启。

2. 原生保留 ARCore 为可选，备用机若有认证机型再验收
   - 保留 ARCore 依赖与代码复杂度；
   - 主展示机仍验收不了 F2；
   - M3D-07（五张触发卡）继续占用建模资源。

3. AR 识别改做微信小程序
   - 是另一个项目（小程序前端 + 后端落地页 + App Link）；
   - 计划书 §3.6 已明确「该能力不属于当前 MVP」；
   - 工作量翻倍，不在当前范围。

## 决定

**归档 F2 触发图入口**。原生 App 不引入 ARCore 依赖，仅保留 F1（二维码）+ F3（手动列表）。

- F2 相关的计划书章节、验收项、任务卡（M3D-07）标记为「归档，可重启」；
- 不删除计划书中 F2 的设计描述（保留为后续重启的依据）；
- 触发图资产（trigger_S1..S5）暂不生产。

二维码入口 F1 继续用计划书原选型 **ZXing Android Embedded 4.3.0**。

## 影响

### 计划书章节调整（标注归档，不改边界）

| 章节 | 原内容 | 调整 |
|---|---|---|
| §1.2 AR 与 VR 边界 | 触发图入口 ARCore | 标注「归档，可重启；MVP 仅二维码+手动」 |
| §2.1 F2 | 触发图识别入口 | 同上 |
| §3.2 技术选型 | ARCore Augmented Images | 同上 |
| §5.3 触发图规范 | arcoreimg 评分、A6 打印 | 归档（M3D-07 暂停） |
| §6.17 权限矩阵 | 触发图需 ARCore | 保留，标注归档 |
| §10.1 | 触发图 10 次识别 9 次、无 ARCore 降级 | 触发图验收项归档；二维码/手动降级验收保留 |
| §13 M3D-07 | 五张触发卡 | 归档 |
| §15.3 MT-001 | 触发图识别率 | 归档 |

### 不变的核心边界（本 ADR 不触及）

- 入口仍只返回 `scene_id`；
- 识别成功后释放相机与 ARCore Session；
- 后续是纯虚拟移动端 VR（Filament + 姿态 + 移动 + 交互 + 音频）；
- 不把 ARCore Pose / Anchor / 现实相机帧带入虚拟展馆。

### 工程影响

- CODE-09（触发图入口）从依赖顺序中移除，改为「归档任务，重启时再实现」；
- App 不声明 ARCore 依赖（`META-INF/com.google.android.ar` AR Optional 标记可保留以备重启，但 minSdkManifest 不强制 ARCore）；
- APK 体积减小（无 ARCore native lib）；
- 主展示机 nubia Z70 Ultra 完整可用（F1 + F3 + 触屏 VR）。

## 回滚方案

重启 F2 时：
1. 确认至少一台 ARCore 认证机型（备用机或低配机）；
2. 在 Version Catalog 加回 ARCore 依赖；
3. 实现 CODE-09（图片库、识别状态、Session 释放）；
4. 建模方启动 M3D-07（触发卡生产与评分）；
5. 恢复 §10.1 触发图验收项与 MT-001。
